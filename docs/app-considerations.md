# 実装時の検討事項 (アプリ側)

テンプレート実装時に洗い出した、アプリケーション側 (frontend / backend) の検討事項です。本テンプレートでは未対応・方針未確定のため、今後の改善対象として GitHub Issue で管理しています (インフラ側の検討事項は各 Issue を参照)。

## 設計・挙動の方針

| 検討事項 | 概要 | Issue |
|---|---|---|
| 排他制御 | 楽観ロック / 重複作成防止 / DB 一意制約の方針整備 | [#144](../../../issues/144) |
| 例外制御・エラー表示 | バックエンド例外制御の方針整備とフロント側エラー表示の一気通貫設計 | [#146](../../../issues/146) |
| 日時・タイムゾーン | JST 決め打ち解消 / 保存=UTC・表示=Intl / DB を Timestamptz 化 | [#148](../../../issues/148) |
| 多言語対応 (i18n) | フロント UI / バックエンドメッセージ / ロケール・日時の扱いの方針検討 | [#147](../../../issues/147) |
| 画面デザイン指針 | CSS 方針の決定と基盤整備 | [#149](../../../issues/149) |
| 環境別の見た目 | 本番 / ステージング / 開発を識別できるよう見た目を変える | [#145](../../../issues/145) |
